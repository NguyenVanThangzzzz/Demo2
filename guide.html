Hướng dẫn Công việc Back-end Developer

Quy trình làm việc hàng ngày

  • 9:00 - Daily Standup Meeting với team
  • Cập nhật Jira/Azure DevOps ticket status
  • Code review với team members
  • 16:30 - Báo cáo tiến độ công việc cho Team Leader

Quy trình phát triển

1. Nhận và phân tích yêu cầu

  • Đọc kỹ requirement từ BA/PM
  • Trao đổi với team để làm rõ yêu cầu
  • Ước tính thời gian thực hiện

2. Thiết kế và lập trình

  • Thiết kế database schema
  • Viết API documentation (Swagger/OpenAPI)
  • Tuân thủ coding convention của team

3. Testing và Review

  • Viết unit test cho code mới
  • Tạo pull request và chờ review
  • Sửa code theo feedback từ reviewer

Công cụ và môi trường làm việc

Development Tools

  • Visual Studio/VS Code
  • SQL Server Management Studio
  • Postman/Swagger UI

Collaboration Tools

  • Git/GitHub/Azure DevOps
  • Jira/Azure Boards
  • Microsoft Teams/Slack

Best Practices

  • Tuân thủ SOLID principles và Clean Code
  • Viết comment và documentation đầy đủ
  • Optimize performance và security
  • Regular backup và version control
